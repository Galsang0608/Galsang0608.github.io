交互题

函数式交互

核心机制

接口调用与实时反馈。评测机与选手代码联合编译。选手仅需实现特定函数，并在函数内部调用交互库提供的 query 等接口获取实时信息，以此完成动态推导。

代码结构

不需要也不可以实现 main 函数，无需写输入输出解绑优化，严格按题意规范和头文件实现指定接口。

易错点

滥用 define int long long 修改函数类型导致编译错误，多测不清空（老生常谈的问题了）。

标准 I/O 交互

核心机制

标准输入输出流在线问答。代码程序与评测机独立运行，双方通过控制台的 cin / cout 进行异步问答。

代码结构

必须写 main 函数，和普通的题目一样。

易错点

每次输出询问参数后，没有用 cout<<endl; 或 cout<<flush; 刷新缓冲区，导致评测机无法接收查询流，直接引发 TLE 或 ILE（得到滚木）。

核心机制
